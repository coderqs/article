## 自定义规范
### 命名规则
4位版本号 + 版本控制号（svn、git）

### 软件版本号说明
```
    ***.***.***.***
     |   |   |   | 
	大   小  BUG 编
    改   改  修  译
    动   动  正  数
```
1. 第一位：大改动（添加了个大的需求）时+1  
2. 第二位：小改动（在原来需求基本不变的情况下，修改部分接口）时+1 
3. 第三位：修改bug时+1  
4. 第四位：每次编译出版本时+1  

## 产品版本号
产品指的是由多个依赖组成的可以部署的完整软件包。
## 产品包命名规则
软件名+4位版本号  
四位版本号：软件版本号前3位+版本控制号

## 产品包内容
1. 软件产品的依赖
2. 配置文件
3. ReadMe（依赖说明）
